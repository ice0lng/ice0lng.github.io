使用反弹shell直接链接数据库，

会发现shell会直接卡在这里不动了。这是因为它只是将bash的输入和输出都重定向到了我们的受控机器，当我们连接mysql的时候，这3306的数据并不会回传过来，导致它一直等待。这时候我们需要一个完全交互的shell。

建立完全交互式的shell

使用python pty

可以一开始时，就创建一个完全交互的shell

使用socat建立完全交互的shell

攻击机开启监听：

使用当前会话的tty终端来作为数据源，使用原始模式（raw）传输数据，不进行回显（echo=0）；

靶机：

exec:’bash -li’：表示在 socat 的连接中使用 bash 进程来执行命令。li 表示使用登录 shell;

pty：表示使用伪终端;

stderr：表示将标准错误输出转发到连接中;

setsid：使得 socat 执行的进程在新 session 中运行，这样可以避免某些信号被 socat 中止;

sigint：将 SIGINT 信号发送到 bash 进程；

sane：将终端设置为“基本模式”（普通终端）；

除了使用shell链接以外，我们还可以使用端口转发，这需要我们能够远程连接mysql，那么需要配置mysql，让其能够被远程访问

mysql远程访问配置

①mysql 8.0以后需要修改 mysqld.cnf 数据库配置

注释下面两条 127.0.0.1的配置，或者将其修改为0.0.0.0

②修改mysql中的mysql库的user表

将上面的User为root的Host修改为%，则可以让mysql的root在任意位置上登陆。

最后mysql中执行：flush privileges;

ssh进行端口转发

做端口转发的工具有很多种，最简单的是使用ssh，比较好用，但链接可能不稳定（可以使用autossh，命令都是一样的）

在需要访问的机子(192.168.100.200)中执行该命令，意思是将访问192.168.100.70:13306端口的数据转发到本机的3306端口中。（一种反向隧道,反向隧道即接收其他机器的数据到本机；正向隧道即将自己的数据转发到其他机器）

然后再在远程主机(192.168.100.70)上运行 mysql -h 127.0.0.1 -P 13306 -uxxx -p 进行连接。
